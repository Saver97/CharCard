## 三、非 Schema 路径的常见错误

以下路径不在 Schema 中，AI 曾错误使用 → 会被拒绝：

| 错误路径 | 应该是 |
|---------|--------|
| `/世界/事件` | 不存在于 Schema |
| `/主角/邦布伙伴` | 应在 `/邦布/邦布伙伴` |
| `/邦布/纸袋布`（作为对象 add） | `add /邦布/拥有的邦布/- "纸袋布"` |
| `/剧情/主线` | 不存在 → 应写入 `/任务/…` |
| `/任务/当前目标` | 应在 `/任务/详情/任务名/状态` → 不存在 |
| `/任务/委托方` | 应在 `委托人` 内 |
| `/物品/未归类/…` | 应写入 `/主角/背包/物品名` |
| `/主角/背包/物品/-` | 背包是对象，不是数组 |
| NPC 着装 key `头部/身体/足部/武器` | 必须用 `上衣/下装/内衣/内裤/鞋子/饰品` |
| NPC 关系 key `关键记忆` | 关系仅 4 字段：好感度/信任度/修罗场/对主角看法 |

## 六、关键修复要点（接续任务时参考）

1. **邦布**：`邦布.邦布伙伴` 是 string（仅 1 只），`邦布.拥有的邦布` 是 string[]，add 用 `/-`，禁止 add 对象到 `/邦布/xxx`
2. **NPC 着装 6 格**：必须是 `上衣/下装/内衣/内裤/鞋子/饰品`，格式 `{名称,品质,描述}`——旧 key `头部/身体/足部` 全部废弃
3. **NPC 关系仅 4 字段**：好感度/信任度/修罗场/对主角看法 —— 禁止 `关键记忆` 等额外字段
4. **背包是对象不是数组**：add 必须 `/主角/背包/物品名 {数量,描述}`，禁止 `/-`
5. **任务路径**：仅接受 `/任务/已激活`（数组）、`/任务/详情/任务名/状态|委托人|报酬|线索`
6. **城市流言**：必须 replace，禁止 add
7. **主角着装 6 格**：同 NPC，string 类型直接 replace 字符串
8. **非法厄同预设服饰**：区分男女（女性填内衣+内裤，男性仅内裤）
9. **输出格式**：使用 `${path}` 占位模板（参考幻器），不是硬编码示例值
