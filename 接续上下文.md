## 一、项目概述

为 SillyTavern 创建《绝区零》RPG 角色卡（支持法厄同/非法厄同双开局 + 完整 MVU 变量系统 + 可视化面板 + EJS 章节沙盒系统）。

**构建命令**：
```bash
cd 绝区零
node C:\Users\Agent\.zcode\skills\tavern-cards\scripts\tavern-cards-forge.mjs pack 绝区零
cd cards\绝区零 && node post-pack.js   # 后处理（必跑）
```

**⚠️ 面板文件路径**：编辑 `面板/` 目录下的 HTML 文件后，必须复制到 `cards/绝区零/正则/` 再打包——forge 读的是正则目录。

```bash
copy /Y 面板\角色创建面板.html cards\绝区零\正则\角色创建面板.html
copy /Y 面板\信息面板.html cards\绝区零\正则\信息面板.html
```

**输出**：`绝区零/cards/绝区零/绝区零.json`（157 条目 + 4 正则 + 2 脚本）

## 四、关键规则（接续任务时注意）

### 修改流程（纯 forge）
```
改 yaml/html → 复制面板到正则目录 → forge pack → post-pack.js
```

### 角色条目格式（角色扮演向，不含游戏数值）
```yaml
name/role/gender/height/type/appearance/personality/weapon/background/info/sp_form
```
禁止：attribute/specialty/rank/CV/版本号/配队/游戏机制

### ⚠️ 禁止编造
所有世界书条目必须基于 wiki 数据，不得自行创作角色细节。已修正的编造案例：普罗米娅（火双剑→黑枝冰异常）、希希芙（人类→蛇希人）、Youkai（编造情感细节）、铅杖（大量编造→仅确认事实）

### 已修复的坑
- 面板JS括号匹配 → 用 `node -e "new Function(js)"` 验证
- `ch.t/m/r` 短名 → `ch.time/money/rumor` 长名（匹配CHAPTER_DATA字段）
- Post-pack将selective全转constant → 改为仅核心组常驻
- 信息面板 `renderBangbooGallery` 报错 → 元素不存在导致catch覆盖所有tab
- 关键词在`keywords`但forge用`strategy.keys` → 已同步
- NPC路径 `/NPCs/-` → NPCs是对象不是数组，必须`/NPCs/{名称}`
- 面板模板字符串单引号不插值 → 改反引号/字符串拼接
- 面板文件BOM字符 → 导致srcdoc解析失败
- 面板onclick未传this → 选中效果不更新
