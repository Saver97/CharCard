# SillyTavern 角色卡构建框架 · 工作流总结

本文档基于绝区零 RPG 项目的实践，抽象出通用的 SillyTavern 角色卡构建框架。**不包含绝区零项目任何具体内容**，仅保留工作流结构与关键模式。

---

## 1. 项目目录结构

```
{项目名}/
├── .cardrc.json                  # 项目元信息 & 构建策略配置
├── build-all.js                  # 主构建脚本（一键生成 JSON）
├── 上下文总结.md                  # 项目上下文（供 AI 续写使用）
├── MVU/                           # 变量系统（MVU = Mag Variable Update）
│   ├── index.js                   # 聚合器：loadAllPanels / generateInitYAML / generateUpdateRules
│   ├── 变量结构.js                 # Zod Schema 定义（ESM，构建时内联到 JSON）
│   ├── {域}面板.js                # 变量面板：世界/主角/空洞/阵营/NPC/邦布 ……
│   └── 角色创建面板.js             # 角色创建变量面板
├── 面板/                          # HTML 面板（由 regex_script 动态注入对话）
│   ├── 角色创建面板.html           # <character_creation> 标签触发
│   └── 信息面板.html               # <StatusPlaceHolderImpl/> 标签触发
├── 开场白/
│   └── 0.txt                     # first_mes 内容
└── cards/{项目名}/
    ├── {项目名}.json              # ★ 最终构建产物（导入 SillyTavern）
    └── 世界书/                    # YAML 格式世界书条目
        ├── 世界观核心/             # 世界基本设定
        ├── 地理与势力/             # 地图、城区、势力分布
        ├── 阵营章节/               # 各阵营组织信息（构建时自动合并角色条目）
        ├── 关键人物/               # 人物 YAML（构建时可合并到阵营，禁用独立条目）
        ├── 扮演准则/               # 叙事基调、规则说明
        ├── 时间线/                  # 历史沿革、剧情节点
        └── MVU/                    # 变量系统 YAML（由 MVU/index.js 自动生成）
            ├── 初始变量.yaml
            ├── 变量列表.yaml
            ├── 变量更新规则.yaml
            ├── 变量输出格式.yaml
            └── 面板_{面板名}.yaml   # 各面板的触发词 + 变量定义
```

## 3. build-all.js — 主构建脚本

### 3.1 职责

- 生成世界书 YAML 文件
- 调用 MVU/index.js 生成变量系统 YAML
- 读取 YAML 目录 → 构建为 SillyTavern `character_book.entries[]`
- 读取面板 HTML → 构建 `regex_scripts[]`（标签替换）
- 组装最终 JSON → 输出到 `cards/{项目名}/{项目名}.json`

### 3.2 核心结构模板

```js
const fs = require("fs");
const path = require("path");
const mvu = require("./MVU/index.js");
const B = path.join(__dirname, "cards", "项目名", "世界书");

// 1. 创建目录结构
const CATEGORIES = ["世界观核心", "地理与势力", "阵营章节", "关键人物", "扮演准则", "时间线", "MVU"];
CATEGORIES.forEach(d => fs.mkdirSync(path.join(B, d), { recursive: true }));

// 2. 内容写入（硬编码 or 从外部数据源读取）
const W = (f, c) => fs.writeFileSync(path.join(B, f), c, "utf8");
W("世界观核心/条目名.yaml", `---\nname: 条目名\nkey: value`);

// 3. MVU 生成（面板 → YAML）
const panels = mvu.loadAllPanels();
W("MVU/变量列表.yaml", mvu.generateVariableList());
W("MVU/初始变量.yaml", mvu.generateInitYAML(panels));
W("MVU/变量更新规则.yaml", mvu.generateUpdateRules(panels));
W("MVU/变量输出格式.yaml", mvu.generateOutputFormat());
panelEntries.forEach(pe => W("MVU/面板_" + name + ".yaml", pe.yaml));

// 4. 开场白
fs.writeFileSync(path.join(__dirname, "开场白/0.txt"), "<character_creation></character_creation>", "utf8");

// 5. YAML → JSON entries
const entries = [];
let uid = 0;
function addYamlDir(dir) {
  // 递归读取所有 .yaml 文件
  // 提取 name → keys, triggers → keys
  // 构建 constant:true, selective:true 条目
  // MVU 文件夹内容 constant:true 但不提取触发词
}
addYamlDir(B);

// 6. 后处理：角色条目合并到阵营章节（仅输出 JSON 生效）
const FOLDER_MAP = { '阵营A':'阵营A', ... };
entries.forEach(e => {
  if (e.extensions.group !== '阵营章节') return;
  const factionName = e.comment.replace('.yaml', '');
  const folder = FOLDER_MAP[factionName];
  if (!folder) return;
  const charEntries = entries.filter(ce => 
    ce.extensions.group === '关键人物' && ce.comment.startsWith(folder + '/')
  );
  if (charEntries.length === 0) return;
  e.content += '\n\n# === 关键人物 ===\n' + charEntries.map(ce => ce.content).join('\n\n');
  charEntries.forEach(ce => { ce.enabled = false; }); // 禁用独立条目
});

// 7. Panel HTML → regex_scripts
const regexScripts = [
  {
    id: "zzz-character_creation",
    scriptName: "初始化面板-角色创建",
    findRegex: "<character_creation>[\\s\\S]*?<\\/character_creation>",
    replaceString: "```html\n" + 角色创建面板HTML + "\n```",
    placement: [2],  // 仅替换用户消息中的标签
    markdownOnly: true, runOnEdit: true
  },
  {
    id: "zzz-StatusPlaceHolderImpl",
    scriptName: "信息面板-状态总览",
    findRegex: "<StatusPlaceHolderImpl/>",
    replaceString: "```html\n" + 信息面板HTML + "\n```",
    placement: [2], markdownOnly: true, runOnEdit: true
  },
  {
    id: "zzz-hide-update",
    scriptName: "隐藏变量更新",
    findRegex: "/<UpdateVariable>([\\\\s\\\\S]*?)<\\\\/UpdateVariable>/g",
    replaceString: "",   // 删除标签，保留内容不显示
    placement: [1, 2],   // 同时处理用户和角色消息
    markdownOnly: true, runOnEdit: true
  }
];

// 8. Schema 动态读取
const schemaContent = fs.readFileSync("MVU/变量结构.js", "utf8");

// 9. 组装 JSON
const card = {
  spec: "chara_card_v3", spec_version: "3.0",
  name: "项目名",
  description: "...",
  first_mes: altGreetings[0] || "",
  tags: [...],
  data: {
    name, description, first_mes, avatar: "none",
    alternate_greetings: altGreetings.slice(1),
    tags, creator: "tavern-cards-forge", character_version: "1.0",
    extensions: {
      depth_prompt: { depth: 4, prompt: "", role: "system" },
      tavern_helper: {
        scripts: [
          { type:"script", name:"mvuzod", content:"import '...';", ... },
          { type:"script", name:"变量结构", content: schemaContent, ... }
        ]
      },
      regex_scripts: regexScripts
    },
    character_book: { entries }
  }
};
fs.writeFileSync(outPath, JSON.stringify(card, null, 2), "utf8");
```

### 3.3 MVU 条目处理规则

| YAML 文件夹 | 注入方式 | 触发词提取 |
|------------|---------|-----------|
| 世界观核心 | constant:true, selective:true | ✅ 提取 name + triggers |
| 地理与势力 | constant:true, selective:true | ✅ 提取 name + triggers |
| 阵营章节 | constant:true, selective:true | ✅ 提取 name + triggers |
| 关键人物 | constant:true, selective:true | ✅ 提取 name + triggers |
| 扮演准则 | constant:true, selective:true, prevent_recursion:true | ✅ 仅 name |
| 时间线 | constant:true, selective:true, prevent_recursion:true | ✅ 仅 name |
| **MVU** | **constant:true, selective:true** | **❌ 不提取触发词** |

## 4. MVU 变量系统

### 4.1 面板定义规范

每个 `MVU/{域}面板.js` 导出以下结构：

```js
module.exports = {
  name: "面板名称",
  description: "面板描述",
  triggers: ["触发词1", "触发词2", ...],
  variables: {
    "路径.键名": { type: "string|number|boolean|array|object", default: 默认值, desc: "描述" },
  },
  variableTemplate: {  // 模板变量（仅注释，不参与初始化）
    "路径.模板键": { type: "string", default: "默认", desc: "描述" },
  },
  rules: [
    "① 规则描述",
  ]
};
```

### 4.2 Schema 定义（变量结构.js）

```js
import { registerMvuSchema } from 'https://testingcf.jsdelivr.net/gh/StageDog/tavern_resource/dist/util/mvu_zod.js';

export const Schema = z.object({
  域1: z.object({
    键1: z.string().or(z.literal('默认值')).prefault('默认值'),
    键2: z.coerce.number().prefault(0),
    键3: z.array(z.string()).prefault([]),
  }).prefault({}),
  
  NPCs: z.record(z.string(), z.object({     // 动态 key 的 record
    基础: z.object({ ... }).prefault({}),
    关系: z.object({ ... }).prefault({}),
    ...
  }).prefault({})).prefault({})
});

$(() => {
  registerMvuSchema(Schema);
});
```

### 4.3 数据流

```
面板 JS (变量定义)
  → MVU/index.js (聚合 → 初始变量.yaml + 更新规则.yaml + 输出格式.yaml)
  → build-all.js (常量注入到 worldbook entries，不提取触发词)
  → SillyTavern 加载 (MVU 插件读取 initvar → stat_data)
  
角色创建面板 HTML (startAdventure)
  → <span style="display:none;"><UpdateVariable><JSONPatch>[...]</JSONPatch></UpdateVariable></span>
  → AI 识别并处理 JSON Patch
  → regex_script "隐藏变量更新" 删除 <UpdateVariable> 标签（内容由 MVU 引擎处理）
  → MVU 引擎解析 JSON Patch → 更新 stat_data
  → eventOn(VARIABLE_UPDATE_ENDED) → 信息面板 renderAll() 刷新
```

### 4.4 JSON Patch 输出格式（世界书注入）

```yaml
---
output_format: |
  <UpdateVariable>
  <Analysis>简述变动因果</Analysis>
  <JSONPatch>
  [
    { "op": "replace", "path": "/stat_data/目标路径", "value": 新值 },
    { "op": "add", "path": "/stat_data/父路径/新键", "value": {...} },
    { "op": "remove", "path": "/stat_data/目标路径" }
  ]
  </JSONPatch>
  </UpdateVariable>
```

## 5. 面板系统

### 5.1 角色创建面板

**触发方式**：开幕 `<character_creation></character_creation>` 自动替换为 HTML 面板。

**核心功能**：
- 开局模式选择（预设/自定义）
- 角色属性填写
- 邦布/伙伴选择（可灰显禁用项）
- `startAdventure()` 一键发送初始化消息 + 变量更新

**变量注入模板**：
```js
const patches = [
  { op: "replace", path: "/世界/当前时间", value: "..." },
  { op: "replace", path: "/主角/基础/姓名", value: name },
  { op: "add", path: "/NPCs/角色名", value: { 基础: {...}, 关系: {...} } },
  ...
];
const patchStr = JSON.stringify(patches);

// ★ 关键：用 span display:none 包裹 UpdateVariable，确保被 regex 删除后不可见
const hiddenDirective = `<span style="display:none;">
<UpdateVariable>
<JSONPatch>
${patchStr}
</JSONPatch>
</UpdateVariable>
</span>`;

const finalPrompt = `/send ${narrative}\n\n${hiddenDirective}|/trigger`;
triggerSlash(finalPrompt);
```

**UI 锁定**：
```js
// 点击开始后禁用所有交互
document.querySelectorAll('input, select, button').forEach(el => el.disabled = true);
document.querySelectorAll('.route-card, .preset-btn, .bb-card').forEach(el => {
  el.onclick = null;
  el.style.pointerEvents = 'none';
  el.style.opacity = '0.6';
});
```

### 5.2 信息面板

**触发方式**：用户手动输入 `<StatusPlaceHolderImpl/>` 标签。

**核心功能**：
- 多页签数据展示（世界观/主角/空洞/阵营/邦布/NPC）
- `getProp(obj, '路径.键', 默认值)` 递归读取嵌套对象
- 事件驱动刷新：`eventOn(Mvu.events.VARIABLE_UPDATE_ENDED, renderAll)`
- Hijack 队列：操作先入 `window.pendingActions[]`，再点击 ✨提交变更 按钮批量发送

**Hijack 队列模板**：
```js
window.pendingActions = [];
function updateHijack(){
  document.getElementById('hc-count').innerText = window.pendingActions.length;
  document.getElementById('hc-submit').disabled = window.pendingActions.length === 0;
}
window.hijackAndSend = function(){
  if(window.pendingActions.length === 0) return;
  const lines = window.pendingActions.map(a => a.desc).join('\n');
  triggerSlash('/send 请执行以下操作：\n' + lines + '|/trigger');
  window.pendingActions = [];
  updateHijack();
};

// 各项操作入队
window.addHeroItem = function(){
  const name = nameEl.value.trim();
  if(!name) return;
  window.pendingActions.push({desc: '添加道具 ' + name, cmd: '/setvar 主角.背包.' + name + ' ' + qty + '|/trigger'});
  updateHijack();
  ...
};
```

**NPC底图居中覆盖**：
```css
.npc-card-w { background-size: cover; background-position: center; }
.npc-bg { position: absolute; inset: 0; background-size: cover; background-position: center; opacity: .15; }
```

**重置底图按钮**：
```js
window.resetNpcBg = function(npcName){
  if(confirm('确定要重置 [' + npcName + '] 的底图吗？')){
    delete window.npcImageCache[npcName];
    try { localStorage.removeItem('zzz_npc_bg_' + npcName); } catch(e) {}
    renderAll();
  }
};
```

### 5.3 面板与 regex_script 对应关系

| HTML 面板 | 触发标签 | regex 模式 | placement |
|-----------|---------|-----------|-----------|
| 角色创建面板 | `<character_creation>...</character_creation>` | 完整标签配对 | [2] (仅用户消息) |
| 信息面板 | `<StatusPlaceHolderImpl/>` | 自闭合简写 | [2] (仅用户消息) |
| 隐藏变量 | `<UpdateVariable>...</UpdateVariable>` | 删除替换（由 MVU 处理） | [1,2] (用户+角色) |

## 6. 世界书条目写入规范

### 6.1 YAML 格式

```yaml
---
name: 条目名称（同时也是触发键）
triggers:
  - "触发词1"
  - "触发词2"
description: 条目描述
key: value
```

### 6.2 条目属性（entries[] 构建）

```js
{
  id: uid,
  keys: ["从 name 和 triggers 提取的触发键"],
  secondary_keys: [],
  comment: "文件名（不含 .yaml）",
  content: "YAML 原始内容",
  constant: true,       // 始终注入
  selective: true,      // 按需触发
  enabled: true,
  position: 'before_char',  // or 'after_char'
  use_regex: true,
  extensions: {
    position: 0,         // 0=before_char, 1=after_char（本例外）, 4=after_char
    depth: 4,
    probability: 100,
    group: "文件夹名",   // 用于后处理过滤
    prevent_recursion: false,  // 扮演准则/时间线 为 true
    role: 0,             // 0=无筛选, 2=角色筛选
    ...
  }
}
```

## 8. 关键设计模式总结

| 模式 | 说明 |
|------|------|
| **constant:true + selective:true** | 所有世界书条目均为常量注入，配合触发键实现按需激活 |
| **MVU 不提取触发词** | 变量条目通过 constant 注入，不参与对话关键词匹配 |
| **span display:none 包裹 UpdateVariable** | JSON Patch 隐藏在不可见元素中，regex 脚本删除标签后自动清空 |
| **角色合并到阵营（仅输出JSON）** | `build-all.js` 后处理：合并内容 + 禁用独立条目；项目 YAML 保持独立 |
| **Schema 动态读取** | `变量结构.js` 由 `fs.readFileSync` 读取，确保 Schema 与源码同步 |
| **Hijack 提交模式** | 面板操作入队列 → 点击提交 → 批量发送 `/send` → AI 处理变量更新 |
| **HTML 面板通过 regex 注入** | 面板 HTML 以 `` ```html `` 代码块形式替换标签，SillyTavern 自动渲染 |
| **文件内嵌 vs 外部引用** | 内容生成逻辑集中在 `build-all.js`；变量系统分离到 `MVU/`；面板分离到 `面板/` |
