## 四、关键规则（接续任务时注意）

### 修改流程（纯 forge）
```
改 yaml/html → forge pack → post-pack.js → git add/commit/push
```
面板 HTML 唯一编辑源：`cards/绝区零/正则/`（无面板/目录）。

### 角色条目格式（角色扮演向，不含游戏数值）
```yaml
name/role/gender/height/type/appearance/personality/weapon/background/info/sp_form
```
禁止：attribute/specialty/rank/CV/版本号/配队/游戏机制

### ⚠️ 禁止编造
所有世界书条目必须基于 wiki 数据，不得自行创作角色细节。已修正的编造案例：普罗米娅（火双剑→黑枝冰异常）、希希芙（人类→蛇希人）、Youkai（编造情感细节）、铅杖（大量编造→仅确认事实）

### 已修复的坑
- **❌ 用 build-all.js 打包 → 丢失 MVU 正则脚本**：build-all.js 硬编码生成正则，不读 state.json，导致 3 条 MVU 必需正则丢失。唯一正确方式：forge CLI
- **❌ forge pack 后跳过 post-pack.js → tavern_helper 序列化错误**：forge 输出 tavern_helper 为 Array `[["scripts",...]]`，post-pack.js 修正为 Object `{scripts:[...]}`。跳过会导致 MVU 完全失效
- **❌ 编辑面板/目录的文件 → 未生效**：forge 读的是 `正则/` 目录。面板/目录已删除，`正则/` 为唯一源
- **❌ 面板 HTML 缺少代码块包裹 → 点击无响应+背板丢失**：`markdownOnly: true` 的正则需要 ```html...``` 代码块包裹，裸 HTML 不渲染。post-pack.js 自动补代码块
- 面板JS括号匹配 → 用 `node -e "new Function(js)"` 验证
- 信息面板 BOM 字符 → 导致 replaceString 开头 `\ufeff`，HTML 无法渲染
- 信息面板 `renderBangbooGallery` 报错 → 元素不存在导致catch覆盖所有tab
- NPC路径 `/NPCs/-` → NPCs是对象不是数组，必须`/NPCs/{名称}`
- 面板模板字符串单引号不插值 → 改反引号/字符串拼接
- 面板onclick未传this → 选中效果不更新
