# 绝区零 RPG 角色卡项目 · 上下文总结

## 项目目标
为 SillyTavern 创建一套《绝区零》RPG 角色卡，支持**法厄同**和**非法厄同**两种开局模式，搭配完整的 MVU 变量系统和可视化面板。

## 变量系统架构

### 数据流
```
角色创建面板（startAdventure）
  → triggerSlash(/send narrative + hidden <UpdateVariable>)
  → AI 回复 + 输出 <UpdateVariable><JSONPatch>...</JSONPatch></UpdateVariable>
  → regex_script "隐藏变量更新" 将 <UpdateVariable> 替换为空
  → MVU 引擎（mvuzod 脚本）解析 JSON Patch → 更新 stat_data
  → 触发 Mvu.events.VARIABLE_UPDATE_ENDED
  → 信息面板 eventOn 监听 → 调用 renderAll() 刷新所有页签
```

### 变量路径规范（全局统一中文键名）

### 变量路径规范（全局统一中文键名）

| 域 | 路径前缀 | 示例 |
|---|---|---|
| 世界观 | `世界.` | `世界.当前时间`, `世界.当前地点`, `世界.城市流言` |
| 主角 | `主角.` | `主角.基础.姓名`, `主角.状态.HP`, `主角.属性.以太适应性`, `主角.当前情绪` |
| 主角着装 | `主角.着装.` | `上衣`, `下装`, `内衣`, `内裤`, `鞋子`, `饰品`（6格） |
| 主角背包 | `主角.背包.` | 键为物品名，值为 `{数量, 描述}` |
| 空洞 | `空洞.` | `空洞.当前空洞`, `空洞.以太浓度`, `空洞.探索进度` |
| 阵营 | `阵营.` | `阵营.狡兔屋`, `阵营.对空六课` |
| 邦布 | `邦布.` | `邦布.伊埃斯电量`, `邦布.邦布伙伴`（string，仅可携带1只）, `邦布.拥有的邦布`（string[]） |
| NPC | `NPCs.` | `NPCs.妮可.基础.性别`, `NPCs.妮可.关系.好感度` |
| NPC 着装 | `NPCs.名称.着装.` | 同主角 6 格：`上衣/下装/内衣/内裤/鞋子/饰品` |

### MVU 条目插入配置
- **position**: `after_char`（extensions.position = 4）
- **depth**: 2（系统插入深度，防止对话增长时被丢弃）
- **apply to**: MVU 文件夹下全部 11 条 YAML

## 面板系统

### 角色创建面板（v2.0）
- **法厄同线**：♂哲 / ♀铃 选择 → 自动配置 Random Play、50000丁尼、以太适应性85、伊埃斯
- **非法厄同线**：3 种预设（普通居民/治安官/盗洞客）+ 完整自定义表单 + 邦布选择
- **"⚡ 开始冒险"**：一键发送初始化消息+变量设置，按钮仅可生效一次
- **邦布记录**：开局邦布同步写入 `邦布.拥有的邦布`

### 信息面板（v4.4）
- **6 个页签**：🌍世界观 / 🐣主角 / 🌀空洞 / 🤝阵营 / 🤖邦布 / 💝NPC
- **邦布**：已拥有邦布从 `邦布.拥有的邦布` 读取；12 个内置邦布图鉴作参考；支持添加新邦布；`邦布.邦布伙伴` 为 string（仅可携带1只，替换时旧邦布自动回列表）
- **主角**：🎒 背包展示与编辑；着装 6 格（上衣/下装/内衣/内裤/鞋子/饰品）模态框选择器
- **NPC**：完全从 MVU 变量读取（无演示数据）
- **事件驱动**：监听 `Mvu.events.VARIABLE_UPDATE_ENDED` 自动刷新
- **数据缓存**：`window.cachedData` 供着装选择器等模态框使用
- **空洞守卫**：`isInHollow()` 守卫 — 空洞中禁止修改邦布

## AI 变量输出规范

### 着装换装规则
- **卸下**：`replace /主角/着装/上衣 ""` + `add /主角/背包/物品名 {数量:1,描述:"..."}`
- **穿上**：`replace /主角/着装/上衣 "新上衣"` + `remove /主角/背包/旧上衣`
- **路径白名单**：`上衣 | 下装 | 内衣 | 内裤 | 鞋子 | 饰品`

### 邦布伙伴规则
- `邦布.邦布伙伴` 是 **string**（仅可携带1只）：`{ "op": "replace", "path": "/邦布/邦布伙伴", "value": "企鹅布" }`
- `邦布.拥有的邦布` 是 **string[]**（可拥有多只）：`{ "op": "add", "path": "/邦布/拥有的邦布/-", "value": "企鹅布" }`

### AI 常见错误
| 错误 | 正确做法 |
|---|---|
| `remove /主角/着装/饰品` | `replace /主角/着装/饰品 ""` |
| `add /背包/物品名`（缺少 `/主角`） | `add /主角/背包/物品名` |
| `邦布.邦布伙伴` value 为字符串 | value 必须为字符串（不是数组） |
| 换装时只 replace 新衣服，旧衣服不入背包 | 同操作中 add 旧衣到背包 |

## 关键修复历程

| 问题 | 解决方案 |
|---|---|
| 信息面板无法显示 | `info_panel` 标签 → `StatusPlaceHolderImpl`；`post_history_instructions` 清空 |
| 开场白过多 | 6 个分散开场白 → 单一 `<character_creation></character_creation>` |
| MVU 条目误触发 | 不提取触发词，纯 `constant: true` 注入 |
| 变量路径不匹配 | MVU 面板 JS、Schema、信息面板、创建面板全部统一为中文键名 |
| Schema 缺字段 | 补充 `当前情绪`、`战斗定位`、`同伴`、`着装`、`背包` |
| 变量结构.js 加载报错 | `MVU/index.js` 跳过该文件（ESM 语法不能 require） |
| MVU 条目被对话长度丢弃 | depth 4→2 + position before_char→after_char |
| 输出格式内嵌 index.js 不便修改 | 抽出为独立 `MVU/变量输出格式.js` 文件 |
| 邦布伙伴数组无法替换 | Schema 从 `string[]` 改为 `string`（仅可携带1只） |
| 邦布伙伴键名含前导空格 | Schema 对齐 `邦布伙伴` 键名 |
| 着装槽位不够精细 | 头部/身体/足部/饰品 → 上衣/下装/内衣/内裤/鞋子/饰品（6格） |
| NPC 着装 Schema 未同步 | Zod enum 同步为 6 格 |

## 使用说明
1. 运行 `node 绝区零/build-all.js` 构建角色卡
2. 在 SillyTavern 中导入 `绝区零/cards/绝区零/绝区零.json`
3. 开场自动弹出角色创建面板，选择开局模式
4. 交互中手动输入 `<StatusPlaceHolderImpl/>` 查看实时状态面板
5. MVU 变量由 AI 回复末尾的 `<UpdateVariable>` 块自动更新
6. 修改变量输出格式 → 编辑 `MVU/变量输出格式.js` → 重新 `node build-all.js`
