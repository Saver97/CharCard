## 信息面板.html v3.x 核心功能

### 6个页签

| 页签 | 内容 |
|---|---|
| 🌍 世界观 | 当前时间/地点/章节/计划 + 天气 + 城市流言 + 绳网活跃度 |
| 🐣 主角 | 身份/生存参数/经济与同伴/⚔️战斗与个性/当前着装(grid卡片可编辑)/背包(可编辑) |
| 🌀 空洞 | 当前空洞/以太浓度/探索进度/研究音擎 + 危险评估(以骸威胁/萝卜质量/安全时间) |
| 🤝 阵营 | 8阵营彩色进度条(正值绿/负值红)+声望称号+委托状态🟢/⚪ |
| 🤖 邦布 | 伊埃斯电量+邦布详情(装备技能/维修状态/心情)+邦布图鉴(12个内置邦布) |
| 💝 NPC | 3个完整演示角色(妮可/艾莲/星见雅)+11个详情区块 |

### 技术特性
- **事件驱动刷新**：Mvu.VARIABLE_UPDATE_ENDED
- **演示模式**：无SillyTavern数据时自动渲染全部演示数据
- **NPC背景图**：IndexedDB持久化存储
- **邦布替换**：点击图鉴中未携带邦布 → 确认 → `/setvar bangboo.邦布伙伴`

## 构建流程

1. 运行 `node build-all.js`：
   - 读取 `MVU/` 下7个面板JS → 生成初始变量YAML + 更新规则 + 面板触发词条目
   - 生成世界书YAML → 102条 world entries
   - 读取 `面板/` 下2个HTML → 生成 `regex_scripts`（角色创建+信息面板）
   - 追加隐藏变量更新 regex_script → `<UpdateVariable>` 替换为空
   - 读取 `开场白/` 下7个txt → alternate_greetings
   - 输出 `cards/绝区零/绝区零.json`（含 `post_history_instructions` 顶层字段）

2. JSON 结构：
   - `spec: chara_card_v3`
   - `post_history_instructions: "[在下一条回复末尾追加显示: <info_panel></info_panel>]"`
   - `data.extensions.regex_scripts` 包含3个条目：
     - `character_creation` → 角色创建面板HTML
     - `info_panel` → 信息面板HTML
     - `隐藏变量更新` → `<UpdateVariable>` 替换为空

## 使用说明

- 运行 `node build-all.js` 构建角色卡
- 在 SillyTavern 中导入 `绝区零/cards/绝区零/绝区零.json`
- 法厄同开局：选择 alternate greetings 0-3
- 非法厄同开局：选择 alternate greetings 4-6（自动弹出角色创建面板，含邦布选择）
- 每次AI回复末尾自动追加 `<info_panel></info_panel>`（通过 `post_history_instructions`）
- 也可手动输入 `<info_panel></info_panel>` 查看实时状态面板
